## Understanding Core Components
The template system uses seven essential building blocks that work together:

1. `glood` - Global store configuration
2. `products` - Product data management
3. `translations` - Multi-language support
4. `section` - Layout and behavior control
5. `template` - Appearance and styling settings
6. `clientInfo` - Device and viewport information
7. `config` - Integration and feature management

## glood Object
The global object contains the store information required by the glood application to work properly on your store.

### Purpose and Usage
The glood object provides essential store information and configuration settings that are used throughout the template. It's particularly important for:
- Building correct store URLs
- Formatting prices according to store settings
- Handling localization
- Managing product references

### Properties Reference

| Key | Type | Description | Required |
|-----|------|-------------|----------|
| `shopify.root_url` | string | Base URL of the Shopify store | Yes |
| `product.id` | string | Current product identifier | Yes |
| `shop.money_format` | string | Store's currency format | Yes |
| `shop.currency` | string | Store's currency code | Yes |
| `localization.language.locale` | string | Language code (e.g., 'en') | Yes |
| `localization.language.primary` | boolean | Whether this is primary language | Yes |
| `localization.language.rootUrl` | string | Base URL for language ('/' for primary) | Yes |
| `localization.country` | string | Country code (e.g., 'IN') | Yes |
| `localization.currency` | string | Currency code (e.g., 'INR') | Yes |
| `market.id` | string | Unique market identifier | Yes |
| `market.handle` | string | Market handle (e.g., 'in') | Yes |

**NOTE:** All property keys are automatically converted to snake_case when used in Liquid templates. For example:
- `rootUrl` → `root_url`
- `devicePixelRatio` → `device_pixel_ratio`
- `moneyFormat` → `money_format`

Example usage in Liquid:
```liquid
{{ glood.shop.money_format }}
{{ glood.localization.language.root_url }}
```

## product Array
The products object is a fundamental component that manages product data and presentation in the template system.

### Purpose and Usage
The products object is responsible for:
- Managing the collection of products to be displayed
- Handling product data organization and presentation
- Controls variant and option configurations
- Managing pricing and inventory information
- Providing media assets for product display

### Properties Reference

| Key | Type | Description | Required |
|-----|------|-------------|----------|
| `id` | string | Product identifier | Yes |
| `title` | string | Product name | Yes |
| `handle` | string | Product URL handle | Yes |
| `vendor` | string | Product vendor name | No |
| `tags` | array | Product tags | No |
| `featured_image.src` | string | Main product image URL | Yes |
| `images[].src` | string | Additional image URLs | No |
| `variants` | array | Product variants | Yes |
| `variants[].id` | string | Variant identifier | Yes |
| `variants[].title` | string | Variant title | Yes |
| `variants[].available_for_sale` | boolean | Stock availability | Yes |
| `variants[].price` | string | Variant price | Yes |
| `variants[].compare_at_price` | string | Original price | No |
| `variants[].image.src` | string | Variant image URL | No |
| `variants[].selected_options` | array | Selected variant options | No |
| `variants[].selected_options[].name` | string | Option name | Yes |
| `variants[].selected_options[].value` | string | Option value | Yes |
| `options` | array | Product options | No |

| `options[].option_values` | array | Available option values | Yes |

**NOTE:** Product array structure is not fixed and it can be changed. If you want to change product array, you can visit the [transformProductData hook](/for-developers/section-template/hooks#transformProductData) in the hooks section.

### Notes on addToCartMode behavior:
1. `card_hover`: Add to cart button slides up from the bottom of the card when hovering over the product card.
2. `image_hover`: Add to cart button appears overlaid on the product image when hovering over the image.
3. `fix`: Add to cart button remains fixed/static at its position.

# css.liquid
The css.liquid file manages all styling for the recommendation sections. It uses Liquid templating to generate dynamic CSS based on your section's settings and configuration.

### Structure and Organization
The CSS is organized by components and breakpoints:
- Base styles for mobile-first approach
- Tablet styles (min-width: 768px)
- Desktop styles (min-width: 1024px)

### CSS Class Reference

| Class Name | Purpose | Properties Controlled | Example Usage |
|------------|---------|---------------------|---------------|
| `._gai-grid` | Main product grid container | - Grid layout (`grid-template-columns`)<br />- Gap between items<br />- Responsive columns | Controls overall product grid layout |
| `._gai-sec-tit` | Section title styling | - Font size<br />- Text alignment<br />- Line clamping<br />- Display properties | Main section heading appearance |
| `._gai-prod-card` | Individual product card | - Background color<br />- Font family<br />- Layout structure | Container for each product |
| `._gai-prod-img` | Product image container | - Aspect ratio<br />- Object-fit<br />- Width/height | Product image display |
| `._gai-prod-tit` | Product title | - Font size<br />- Line clamping<br />- Text alignment<br />- Font weight | Product name styling |
| `._gai-prod-vndr` | Vendor name | - Font size<br />- Display properties<br />- Position | Brand/vendor name display |
| `._gai-prod-prc` | Product price | - Color<br />- Font family<br />- Font size | Main price display |
| `._gai-prod-prc--original` | Compare-at price | - Color<br />- Display<br />- Text decoration | Original/compare price styling |
| `._gai-atc-btn` | Add to cart button | - Background color<br />- Text color<br />- Border radius<br />- Hover states | Add to cart button styling |
| `._gai-dis-cnt` | Discount container | - Justification<br />- Background<br />- Display properties | Discount information wrapper |
| `._gai-dis-txt` | Discount text | - Color<br />- Font size<br />- Text alignment | Discount amount/percentage text |
| `._gai-dis` | Discount badge | - Background color<br />- Text color<br />- Opacity<br />- Position | Discount label styling |

### Label Classes
Special classes for product status and badges:

| Class Name | Purpose | Customizable Properties |
|------------|---------|------------------------|
| `._gai-oos` | Out of stock badge | Background/text colors |
| `._gai-bestseller` | Bestseller badge | Background/text colors |
| `._gai-new` | New product badge | Background/text colors |
| `._gai-trending` | Trending item badge | Background/text colors |

### Responsive Behavior
The file includes media queries for different screen sizes:
```css
/* Mobile (default) */
[data-gai-section-id="{{ section_id }}"] ._gai-grid {
  grid-template-columns: repeat({{- settings.breakpoints.small.cards_per_view -}}, 1fr);
  gap: {{ settings.breakpoints.small.gutter }}px;
}

/* Tablet */
@media (min-width: {{ settings.breakpoints.medium.screen_size }}px) {
  /* Tablet-specific styles */
}

/* Desktop */
@media (min-width: {{ settings.breakpoints.large.screen_size }}px) {
  /* Desktop-specific styles */
}
```

## theme.js
The theme.js file is the core JavaScript file that makes your recommendation sections interactive. It exports an object with methods that handle various section behaviors and data transformations.

### Core Methods

| Method Name | Parameters | Description | Return | Required |
|-------------|------------|-------------|---------|----------|
| `initSwiper` | `(Swiper, templateSettings, container, params, defaultSettings, utils)` | Initializes carousel functionality | `Swiper instance` | Yes |
| `onSectionInit` | `(args, cb, gloodUtils)` | Called when section initializes | `void` | Yes |
| `onSectionRenderComplete` | `(args, utils)` | Called after section rendering completes | `void` | Yes |
| `onVariantChange` | `(args, utils)` | Handles variant selection changes | `void` | Yes |
| `onSwatchChange` | `(args, utils)` | Handles swatch selection changes | `void` | Yes |
| `validateCartOperation` | `(payload, recommendation, utils, gloodUtils)` | Validates cart operations before execution | `{payload, error, href}` | Yes |
| `onCartOperationComplete` | `(args, utils, gloodUtils)` | Called after cart operation completes | `void` | Yes |
| `onProductDataReceived` | `(args, utils)` | Handles incoming product data | `void` | Yes |
| `getProductDetailsQuery` | `(recommendation, glood)` | Returns GraphQL query for product details | `string` | Yes |
| `transformProductData` | `(product, recommendation)` | Transforms raw product data into required format | `object` | Yes |

### Event System

| Event Name | Parameters | Description | Example Usage |
|------------|------------|-------------|---------------|
| `variantChange` | `{productId, variantId, options}` | Triggered when variant is selected | Update prices, images |
| `addToCart` | `{variantId, quantity}` | Triggered when adding to cart | Show cart feedback |
| `imageHover` | `{productId, imageUrl, index}` | Triggered on image hover | Switch product image |
| `imageReset` | `{productId}` | Triggered when hover ends | Reset to main image |
| `carouselChange` | `{index}` | Triggered on slide change | Update navigation |
| `carouselEnd` | `{}` | Triggered at last slide | Load more items |

### Swiper Configuration
The `initSwiper` method handles carousel setup with these parameters:

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `templateSettings` | object | Section template settings | `{carousel: {}, breakpoints: {}}` |
| `container` | HTMLElement | Carousel container element | `document.querySelector('._gai-crz-cnt')` |
| `breakpoints` | object | Responsive breakpoint settings | `{768: {slidesPerView: 3}}` |
| `swiperConfig` | object | Swiper library configuration | `{loop: true, pagination: {}}` |

### Product Data Structure
The `transformProductData` method transforms product data into this structure:

| Field | Type | Description |
|-------|------|-------------|
| `featuredImage` | object | `{ src: string }` Main product image |
| `handle` | string | Product URL handle |
| `id` | number | Product ID (parsed from Shopify GID) |
| `title` | string | Product title |
| `vendor` | string | Product vendor name |
| `images` | array | Array of `{ src: string }` objects |
| `variants` | array | Array of variant objects |
| `available` | boolean | Overall product availability |
| `compareAtPrice` | string | Minimum compare-at price |
| `price` | string | Minimum regular price |
| `variantsCount` | number | Total number of variants |
| `tags` | array | Product tags |
| `availableForSale` | boolean | Whether product can be purchased |
| `options` | array | Product options configuration |

### Variant Data Structure
Each variant in the variants array contains:

| Field | Type | Description |
|-------|------|-------------|
| `id` | number | Variant ID (parsed from Shopify GID) |
| `compareAtPrice` | string | Original price (if discounted) |
| `price` | string | Current price |
| `image` | object | `{ src: string }` Variant-specific image |
| `currentlyNotInStock` | boolean | Stock status |
| `title` | string | Variant title |
| `quantityAvailable` | number | Available quantity |
| `availableForSale` | boolean | Purchasability status |
| `selectedOptions` | array | Selected variant options |

### GraphQL Query Fields
The `getProductDetailsQuery` method requests these fields:

| Field | Subfields | Description |
|-------|-----------|-------------|

| `priceRange` | `minVariantPrice` | Price information |
| `compareAtPriceRange` | `minVariantPrice` | Compare-at prices |
| `options` | `name, optionValues` | Product options |
| `variants` | Multiple fields | Complete variant data |
| `images` | `url` | Product images |
| `vendor` | - | Vendor name |
| `variantsCount` | `count` | Number of variants |
| `tags` | - | Product tags |
