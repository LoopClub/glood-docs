## 1. money
**Description:**  
Formats numeric values into the store's currency format based on the money_format setting from the glood object.

**Parameters:**  
- `value` (Number/String): The numeric price value to be formatted
- `money_format` (String, optional): Override default money format

**Returns:**  
String - Formatted price string with currency symbol

**Example:**
```liquid
{{ 19.99 | money }}  →  "$19.99"
{{ product.price | money: money_format: shop.money_format }}  →  "$29.99"
```

**Use Cases:**
- Formatting product prices
- Displaying compare at prices
- Showing discount amounts
- Bundle total prices

## 2. optimize_image
**Description:**  
Optimizes and resizes images for better performance using Shopify's CDN parameters.

**Parameters:**  
- `url` (String): The original image URL to be optimized
- `width` (Number, optional): Desired image width
- `height` (Number, optional): Desired image height

**Returns:**  
String - Optimized image URL with CDN parameters

**Example:**
```liquid
{{ product.featured_image | optimize_image }}
{{ product.featured_image | optimize_image: width: 300, height: 300 }}
```

**Use Cases:**
- Product card images
- Optimizing featured images
- Responsive image loading
- Performance optimization

## 3. tracking_url
**Description:**  
Adds tracking parameters to product URLs for analytics and attribution.

**Parameters:**  
- `product` (Object): The product object
- `section` (Object): The section context
- `ref_product_id` (String, optional): Reference product ID

**Returns:**  
String - Product URL with tracking parameters

**Example:**
```liquid
{{ product | tracking_url: section }}
{{ product | tracking_url: section, ref_product_id: '123' }}
```

**Use Cases:**
- Product link tracking
- Analytics integration
- Click attribution
- Performance monitoring

## 4. section_discount
**Description:**  
Formats section-level discount information based on the store's locale and money format.

**Parameters:**  
- `section` (Object): The section object containing discount information
- `money_format` (String): Store's money format
- `locale` (String): Current locale

**Returns:**  
String - Formatted discount text

**Example:**
```liquid
{{ section | section_discount }}
{{ section | section_discount: money_format: shop.money_format, locale: 'en' }}
```

**Use Cases:**
- Section headers
- Discount badges
- Sale announcements
- Promotional displays

## 5. color
**Description:**  
Maps color names to valid CSS colors using the template's color mapping settings.

**Parameters:**  
- `color_name` (String): Color name to map
- `settings` (Object): Template settings containing color mappings

**Returns:**  
String - Valid CSS color value

**Example:**
```liquid
{{ variant.color | color: template.settings }}
```

**Use Cases:**
- Color swatch display
- Product variant colors
- UI element styling
- Background colors

## 6. discount_text
**Description:**  
Combines product and section discount information into a formatted string.

**Parameters:**  
- `product` (Object): Product data
- `section` (Object): Section configuration
- `translations` (Object): Translation strings
- `money_format` (String): Shop's money format

**Returns:**  
String - Formatted discount text

**Example:**
```liquid
{{ '' | discount_text: product, section, translations, money_format: shop.money_format }}
```

**Use Cases:**
- Product card discounts
- Sale badges
- Promotional text
- Discount highlights

## 7. savings_text
**Description:**  
Calculates and formats the savings amount for a product.

**Parameters:**  
- `product` (Object): Product data
- `section` (Object): Section configuration
- `translations` (Object): Translation strings
- `money_format` (String): Shop's money format

**Returns:**  
String - Formatted savings text

**Example:**
```liquid
{{ '' | savings_text: product, section, translations, money_format: shop.money_format }}
```

**Use Cases:**
- Savings calculations
- Sale promotions
- Discount displays
- Price comparisons

## Best Practices

1. Always provide required parameters for each filter
2. Use appropriate image dimensions for different viewports
3. Test money formatting with different currencies and formats
4. Consider performance impact of filters
5. Handle empty or invalid values gracefully
